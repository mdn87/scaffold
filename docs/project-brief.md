# Project Brief

## One-Paragraph Summary

Describe the project in plain language. Include what it should do, who it is for, and why it matters.

## Users And Operators

- Who will use it?
- Who will maintain or operate it?

## Core Capabilities

- What should this project be able to do first?
- What would make the first version useful?

## Constraints And Unknowns

- What is still unverified?
- What technical, hardware, budget, or time constraints matter?

## Hardware / Runtime Notes

- Required devices, machines, or operating systems
- Local versus remote execution assumptions
- Any diagnostics or compatibility checks that must happen first

## External Systems

- APIs, services, tunnels, dashboards, printers, microcontrollers, or other dependencies

## First Slice

- What is the safest smallest version to prove the concept?
- What should wait until later?

## Success Checks

- How will we know this project is working well enough to continue?
